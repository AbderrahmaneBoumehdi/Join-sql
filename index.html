Une jointure SQL permet de combiner les lignes de deux ou plusieurs tables en fonction d’une condition commune (généralement une clé étrangère). Cette présentation interactive vous montre exactement ce qui se passe à chaque fois, avec des diagrammes de Venn en cercles et des tableaux SQL réels.

1

INNER JOIN

Seules les correspondances communes

Retourne uniquement les lignes qui ont une correspondance dans les DEUX tables

Table Clients (gauche)

3 lignes
id nom
1Alice
2Bob
3Charlie

Table Commandes (droite)

3 lignes
id_client produit
1Ordinateur
2Souris
4Clavier
→
RÉSULTAT INNER JOIN SELECT c.nom, co.produit FROM clients c INNER JOIN commandes co ON c.id = co.id_client
nom produit
AliceOrdinateur
BobSouris

→ Seules les lignes dont l’id existe dans les deux tables sont conservées (2 résultats).

2

LEFT OUTER JOIN

Tout à gauche + correspondances à droite

Retourne TOUTES les lignes de la table de gauche + les correspondances de la droite (NULL sinon)

Table Clients (gauche)

3 lignes
id nom
1Alice
2Bob
3Charlie

Table Commandes (droite)

3 lignes
id_client produit
1Ordinateur
2Souris
4Clavier
→
RÉSULTAT LEFT OUTER JOIN SELECT c.nom, co.produit FROM clients c LEFT OUTER JOIN commandes co ON c.id = co.id_client
nom produit
AliceOrdinateur
BobSouris
CharlieNULL

→ Charlie n’a pas de commande → produit = NULL

RIGHT OUTER JOIN

Tout à droite + correspondances à gauche

Retourne TOUTES les lignes de la table de droite + les correspondances de la gauche (NULL sinon)

Table Clients (gauche)

3 lignes
id nom
1Alice
2Bob
3Charlie

Table Commandes (droite)

3 lignes
id_client produit
1Ordinateur
2Souris
4Clavier
→
RÉSULTAT RIGHT OUTER JOIN SELECT c.nom, co.produit FROM clients c RIGHT OUTER JOIN commandes co ON c.id = co.id_client
nom produit
AliceOrdinateur
BobSouris
NULLClavier

→ Le clavier n’a pas de client correspondant → nom = NULL

4

CROSS JOIN

Produit cartésien (toutes les combinaisons)

Aucune condition → chaque ligne de la gauche avec chaque ligne de la droite

Table Clients (gauche)

3 lignes
id nom
1Alice
2Bob
3Charlie

Table Commandes (droite)

3 lignes
id_client produit
1Ordinateur
2Souris
4Clavier
✕
RÉSULTAT CROSS JOIN SELECT c.nom, co.produit FROM clients c CROSS JOIN commandes co
nom produit
AliceOrdinateur
AliceSouris
AliceClavier
BobOrdinateur
BobSouris
BobClavier
CharlieOrdinateur
CharlieSouris
CharlieClavier

→ 3 clients × 3 commandes = 9 lignes. Aucune condition de jointure.
